Glossary — key terms (click to expand)
Agent Loop = The cycle: send message to LLM → LLM picks a tool → run tool → send result back → repeat until done.
ReAct = Reasoning + Acting. The LLM reasons ("I need to search"), then acts (calls search tool), then reasons again from the result.
Self-review = Agent checks its own output before responding. Like proofreading. One LLM call.
Self-healing = Agent detects it crashed or got an error and retries automatically. No human needed.
Self-improving = After finishing a task, agent saves what it learned as a reusable "skill." Gets smarter over time across sessions.
Background thread = A second process running in parallel. Hermes spawns a second LLM call in the background after every 10 turns.
Skill file = A Markdown file saved to disk (~/.hermes/skills/) that describes how to do something. Loaded into context next time.
Skill (Runnable/V4) = A manually-written instruction block loaded into the agent's prompt at startup. NOT auto-generated.
Tool = A function the AI can call (read file, run code, search web, etc.).
Context window = The AI's working memory for one session. Cleared when session ends.
Persistent memory = Memory saved to disk/database that survives across sessions. Hermes has this. Standard coding agents don't.
Oracle = Your self-improving sales AI. Right now it uses LLM per call. The goal: make it learn from past calls like Hermes.

The Core Question: What Is Different?

What is a self-improving agent? A normal AI agent (like Claude Code or ChatGPT) answers your question and forgets you exist. A self-improving agent does the same thing, but after answering it asks itself: "Did I learn something new? Should I write it down?" If yes, it saves a note to a file on disk. Next time you ask a similar question, it reads that note first — so it answers faster and better. Over weeks and months, it builds a personal library of everything that worked. That is the difference.
One sentence answer: A coding agent loop runs once per task and forgets everything. A self-improving loop runs forever, and after each task it saves what it learned so the NEXT task goes better.

The Three Loop Types — What, Why, How

What is a loop? An AI agent doesn't just answer once — it runs a cycle: think → use a tool → look at the result → think again → repeat until done. That cycle is the "loop." The type of loop determines whether the agent is a one-time worker or something that gets smarter over time.

Loop 3 — Self-Healing: "Recover from errors automatically"

What is self-healing? When a tool fails (e.g., permission denied, API timeout), the error is returned as a message back to the LLM — NOT as a crash. The LLM reads the error and decides what to try next. It "heals" itself within the session. Both Loop 1 and Loop 2 have this.

What, Why, How — Self-Healing

AspectDetail
What Tool errors are caught and returned to the LLM as a role: tool message with the error text. The LLM sees the error just like it sees any other tool result and decides the next action.
Why Makes agents robust without extra code. The LLM's reasoning handles retry logic naturally — no need to write try/except/retry for every tool. The LLM is smarter than any hardcoded retry strategy anyway.
How In code: try: result = run_tool(...) except Exception as e: result = f"Error: {e}" — that's it. The error string goes back into the message array as if it were a normal result. The loop continues. Hermes also adds: API key rotation on 429 errors, stream retry on network drops.
Limitation Only works within one session. The agent forgets the error when the session ends and will try the same broken approach next time — unless Hermes saved the lesson as a skill.
Self-healing ≠ Self-improving. Self-healing = "I made a mistake, let me retry differently right now." Self-improving = "I made a mistake, let me write it down so I never make it again." The first is a single session. The second is permanent.

Skills: Auto-Generated vs Manually Written

What is a skill? A skill is a text file (Markdown) that tells the agent HOW to do something. Think of it like a recipe card in a kitchen. Instead of figuring out how to make pasta from scratch every time, the chef reads the recipe card and follows it. In AI agents, a skill file says things like: "When the user asks to set up Docker, do these 4 steps in this order. Watch out for this common error." The agent reads the skill before starting work.

The biggest practical difference between Hermes and coding agents: who writes these recipe cards.

What, Why, How — Skills

AspectHermes Auto SkillsV4 / Runnable Manual Skills
What After a complex task (5+ tool calls), a background LLM writes a SKILL.md file to ~/.hermes/skills/. Next session, that file is automatically loaded into the system prompt. A developer (you) writes a .md file by hand and commits it to git. It's loaded at startup every session — always the same until you update it.
Why Zero effort scaling. Every complex task produces a new skill. After 100 sessions, you have 100 skills — all from real experience, not guesswork. The agent gets faster over time without you doing anything. Higher reliability. A human-reviewed skill won't contain hallucinated steps. But it doesn't scale — you can only write so many manually, and they go stale without updates.
How updates work When the agent uses a skill and hits a problem NOT covered by it, the background LLM patches the skill immediately. Self-correcting. You notice the skill is outdated, open the file, edit it, commit, push. Manual process.
Risk Can save wrong patterns or hallucinate steps. No human review. Human-verified. More reliable but slower to grow.

- One-off request unlikely to repeat

Memory: How Each Agent Remembers

What is agent memory? When you talk to ChatGPT, each conversation starts fresh — it doesn't know what you said yesterday. That's "no memory." Agent memory means saving information to a file or database so the AI can read it next time. There are two types: in-session (remembers within one conversation — all agents have this) and cross-session (remembers across conversations — only Hermes has this). Cross-session memory is the key difference.
Simple rule: Coding agents (V4, Runnable) remember nothing between sessions. Hermes remembers everything, forever, across all sessions.

What, Why, How — Memory

AspectHermesCoding Agents
What 5-layer memory: in-session history + cross-session files (~/.hermes/memory/) + skills + user model (Honcho) + memory compression. Full-text search (FTS5) across all past sessions. 1-layer: in-session conversation history in the context window. When it fills up, old turns are compressed into a summary. Session ends = everything gone.
Why it matters Remembering "you prefer dark mode" or "last time Docker failed because of nginx permissions" means the agent never asks the same question twice and avoids past mistakes. Fine for one-off tasks. But if you use the same agent daily, you're re-teaching it every single session. Wasteful.
How it's stored JSON files on disk + SQLite FTS5 index. Background LLM periodically compresses old memories to save space while keeping the essentials. Nowhere. The conversation is in RAM only. When the process exits, it's gone.

The 5 steps of the self-improving loop in code:
1. Counter — count how many turns have passed
2. Spawn — fork a second LLM in a background thread
3. Prompt — tell that LLM "review this conversation, save anything useful"
4. Write — save the skill to a .md file on disk
5. Load — next session, inject all saved skills into the system prompt

Step 2: Spawning the Background Review

run_agent.py:1814 — runs in a daemon thread, invisible to user

def _spawn_background_review(self, messages_snapshot, review_memory, review_skills):
    def _run():
        # Create a SECOND complete LLM agent
        review_agent = AIAgent(
            model=self.model,
            max_iterations=8,
            quiet_mode=True,              # silent, user doesn't see it
        )
        review_agent._memory_nudge_interval = 0   # no recursive nudges!
        review_agent._skill_nudge_interval = 0

        # Give it the SKILL REVIEW prompt + full conversation
        review_agent.run_conversation(
            user_message=_SKILL_REVIEW_PROMPT,
            conversation_history=messages_snapshot,  # full context
        )

    # Run in background thread (non-blocking)
    t = threading.Thread(target=_run, daemon=True)
    t.start()
